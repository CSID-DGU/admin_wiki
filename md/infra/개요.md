# 개요

> **한 줄 요약** — config-server는 "관리자가 웹에서 승인 버튼을 누르면, GPU 컨테이너와 그에 필요한 계정·홈 폴더·접속 포트를 실제로 만들어 주는 서버"입니다.

---

## 1. config-server가 하는 일 (한눈에)

DECS는 연구실 GPU 서버를 웹으로 신청하고 승인받아 쓰는 시스템입니다. 서버 세 개가 역할을 나눠 맡습니다.

| 서버 | 역할 | 비유하면 |
|------|------|----------|
| admin_fe | 사용자·관리자가 보는 웹 화면 | 주문서 접수 창구 |
| admin_be | 신청을 저장하고 승인 여부를 결정 | 주문을 승인하는 매니저 |
| **config-server (이 저장소)** | 승인된 내용대로 실제 자원을 만들고 지움 | 주방 — 실제로 만드는 곳 |

기억할 원칙은 하나입니다. **결정은 admin_be, 실행은 config-server.** config-server는 스스로 판단하지 않고, admin_be가 시키는 대로 Kubernetes와 저장소(NAS)를 조작하는 유일한 창구입니다.

---

## 2. 전체 흐름 (5단계)

```mermaid
flowchart LR
    A[사용자<br/>웹에서 신청] --> B[관리자<br/>승인 버튼]
    B --> C["config-server<br/>계정·홈·컨테이너·포트 생성"]
    C --> D[사용자에게<br/>접속 안내 메일]
    D --> E[만료되면<br/>자동 회수]
```

승인 버튼 하나에 config-server가 실제로 하는 일은 다음 순서입니다.

1. **계정 만들기** — 리눅스 계정(UID)을 공용 계정 파일에 등록합니다.
2. **홈 폴더 만들기** — NAS(공용 저장 장비)에 그 사용자의 홈 디렉터리를 만듭니다.
3. **GPU 컨테이너 띄우기** — 가장 한가한 GPU 서버를 골라 컨테이너(Pod)를 실행합니다.
4. **접속 포트 열기** — SSH·Jupyter로 들어올 수 있는 포트를 배정합니다.
5. **결과 돌려주기** — 배정된 포트를 admin_be에 알려 주고, admin_be가 사용자에게 메일을 보냅니다.

만료가 되면 같은 체인을 거꾸로 돌려 컨테이너·포트·계정을 자동으로 지웁니다.

---

## 3. 왜 자동으로 만들까

이 시스템이 없던 시절에는 관리자가 서버에 접속해 계정·컨테이너·포트를 전부 손으로 만들었습니다. 그 방식의 문제는 세 가지였습니다.

- 신청이 몰리면 **관리자 한 사람이 병목**이 됩니다.
- UID 중복·포트 충돌 같은 **수작업 실수**가 끼어듭니다.
- 만료된 자원을 사람이 기억해서 지워야 해서 **GPU가 방치**됩니다.

config-server는 이 수작업 체인을 통째로 대신하는 "실행자"입니다. 더 자세한 배경과 설계 이유는 [시스템 아키텍처](design/시스템-아키텍처.md)에 있습니다.

## 4. 이 저장소에 뭐가 있나

| 디렉터리 | 내용 | 상태 |
|----------|------|------|
| `config-server/` | Flask API 서버 본체 + Helm 차트 | **핵심** |
| `infra-sql/` | 포트 배정 장부(어떤 포트를 누가 쓰는지 기록하는 표)를 담는 MySQL 배포 파일 | 배포 중 |
| `pvc-image-chart/` | 사용자 이미지 저장소(PVC) 차트 | 배포 중 |
| `pvc-chart/`, `auth-server/`, `containerssh/`, `guestImage/` | 과거 방식의 잔재 | 레거시 — [배포본과 레거시 경계](design/배포본과-레거시-경계.md) |

## 5. 서버 환경 요약

| 항목 | 값 |
|------|-----|
| 배포 위치 | FARM Kubernetes, namespace `ailab-infra` |
| 외부 접근 | NodePort **30082** (Service `containerssh-config-service`) |
| Swagger UI | `http://210.94.179.18:30082/apidocs/` (내부망 전제) |
| 전용 DB (config-server가 관리) | infra-mysql `pod_port_db` — [데이터베이스](design/데이터베이스.md) |

> ℹ️ repo README의 NodePort 9732 표기는 오기입니다. 실측 기준 30082가 맞습니다.

⚠️ config-server API에는 인증이 없습니다. 외부에서 직접 호출할 수 있으면 그대로 인프라 조작이 되므로, 내부망 제한이 전제 조건입니다.
